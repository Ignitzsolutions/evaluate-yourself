# UI Acceptance Criteria

Before a route is considered revised:

1. The route has one clear primary action.
2. Secondary actions are visually quieter.
3. Loading, empty, error, and success states are handled when reachable.
4. User input is preserved after recoverable failures.
5. Keyboard focus is visible.
6. Interactive cards use semantic buttons or links.
7. Mobile layout works at 320px width.
8. No horizontal scrolling is introduced.
9. No arbitrary colors or spacing are added outside the theme.
10. No emojis are used as interface icons.
11. API and AI-generated data is validated or safely bounded before rendering.
12. Relevant tests or build checks pass.
